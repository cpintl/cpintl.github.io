## 🤖 Automated Features

This profile uses **GitHub Actions** for automation:

### Daily Refresh
- **When:** Every day at 9 AM UTC
- **What:** Updates the README timestamp
- **File:** `.github/workflows/update-readme.yml`

### Daily Insight
- **When:** Every day at 7 AM UTC
- **What:** Updates the daily inspirational quote
- **File:** `.github/workflows/daily-quote.yml`

**No manual maintenance needed!** Everything runs automatically.

## 🔧 GitHub Actions Workflows

### View Workflow Status

1. Go to the **Actions** tab in the repository
2. You'll see a list of all workflow runs
3. Green ✅ = Success | Red ❌ = Failed

### Manually Run a Workflow

1. Go to **Actions** tab
2. Click the workflow name (e.g., "Update Profile README")
3. Click **Run workflow** button
4. Check back in 30 seconds to see results

### Disable a Workflow

1. Go to **Actions** tab
2. Click the workflow
3. Click **⋯** (three dots) menu
4. Click **Disable workflow**

## ❓ Frequently Asked Questions

### Q: Why do I need a profile repository?

**A:** GitHub allows you to create a special README that displays on your profile page. It's a great way to:
- Showcase your expertise
- Link to your projects
- Provide contact information
- Share your mission and philosophy

### Q: How do the workflows work?

**A:** GitHub Actions are automation scripts that run on a schedule. They:
- Run automatically at set times (daily, weekly, etc.)
- Make changes to files automatically
- Push those changes to your repository
- Update your profile without manual work

### Q: Can I customize the workflows?

**A:** Yes! Edit the `.github/workflows/` YAML files to:
- Change the schedule (cron times)
- Add new quotes
- Modify what gets updated
- Add new automated tasks

### Q: How do I add a custom domain?

**A:** If you want your profile at a custom domain (e.g., ariful.dev):
1. Buy or have a domain
2. Go to repo **Settings → Pages**
3. Add your custom domain
4. Update DNS records (instructions provided)
